# InstaBot 用户使用说明

这份说明给**每天用 InstaBot 处理 Instagram 私信的人**看。如果你是部署/运维角色，请看 [`OPERATIONS.md`](OPERATIONS.md) 和 [`DEPLOY.md`](DEPLOY.md)。

---

## 1. InstaBot 是什么

InstaBot 是一个 Instagram 私信自动回复 + 客服管理后台。简单说，它能帮你做三件事：

- **AI 自动回复客户私信**：客户发来消息，AI 按你的知识库（价格、覆盖区、时间等）自动回答，省去你手工回复的时间
- **评论触发自动私信**：客户在你帖子下评论关键词（比如「price」），系统自动公开回复 + 发私信
- **后台手动接管**：随时切换到「我亲自接」模式，AI 就停下来不再回复，你自己回；写中文草稿可以让 AI 帮你翻译润色成专业英文

登录后默认进入**仪表盘（Dashboard）**页面。

## 4. 对话页面（核心）

这是你 90% 时间会待的页面。打开后是**两栏布局**：

- **左侧**：对话列表，按最新消息时间排序，每条显示客户头像/用户名/最后一条消息预览
- **右侧**：选中对话的详情 + 输入面板

### 4.1 AI 自动 vs 我亲自接

每个对话都有一个**模式开关**，在对话顶部右侧：

- **AI 自动**（默认）：客户发消息后，AI 会**自动**生成回复并发出去，你只需要旁观
- **我亲自接**：AI 不再自动回复，等你手动处理

**什么时候切到「我亲自接」？**

- 客户问的问题超出知识库范围（比如要求定制服务）
- 客户表达了下单意愿，需要你确认细节
- AI 回答有误，需要你纠正
- 客户情绪激动需要安抚

切换之后 AI 立即停止自动回复这个对话，不会再插嘴。

> 想知道**新客户**默认进哪种模式？去**系统设置 → 新对话默认模式**调整。

### 4.2 让 AI 帮我生成回复（AI 模式下）

如果你切到「AI 自动」但想自己**预览/修改**一下 AI 准备发的内容：

1. 在右下角输入面板里，AI 模式会显示**「生成回复」**按钮
2. （可选）在上方提示词框里加一句风格要求，比如：
   - `回复短一点`
   - `语气更亲切`
   - `不要用感叹号`
3. 点「生成回复」
4. AI 输出的中文草稿会出现在预览区
5. 你可以直接编辑这段内容
6. 点 **「发送 AI 回复」** 才会真的发出去

**重要：** 这里输入的「风格提示词」会被 AI **自动学习成长期偏好**——下次它会自动遵循。比如你提示了 3 次「回复短一点」，以后默认就短了。如果想看/修改/删除已学到的偏好，去**系统设置 → 管理者偏好**。

### 4.3 AI 翻译并优化（人工模式下）

切到「我亲自接」后，输入面板长这样：

```
[输入框：写你的中文草稿]
[ AI 翻译并优化 ]  [ 发送 ]
```

**典型场景**：你想用中文思考一下怎么回客户，然后让 AI 帮你翻译成专业英文。

操作步骤：

1. 在输入框写中文草稿，比如：「我们小件的话每个月100单以下是8块一单，超过的话可以谈一个更优惠的价格」
2. 点 **「AI 翻译并优化」** 按钮
3. AI 会在上方显示翻译后的英文版本
4. 你可以：
   - 点「使用优化版本」→ 把英文版本填进输入框
   - 点「取消」→ 关掉预览，保留原中文
   - 直接点「发送」→ 把当前输入框里的内容发出去

**拖动调整窗口大小：** 翻译预览框和输入框之间有一条蓝色拖条，可以上下拖动调整两个区域的比例。整个面板顶部还有一条灰色拖条，可以调整面板总高度。

### 4.4 发送消息

不管是 AI 模式生成的回复、还是人工模式手写的内容，点**发送**后系统会：

1. 根据**回复语言**设置（系统设置里）决定要不要翻译
2. 调用 Instagram API 发出去
3. 不管发不发得出去，都会**保存到对话历史**

**如果发不出去**，会弹一个橙色提示：「消息已保存，但未能发送到 Instagram」。常见原因看 [4.6](#46-头像和状态显示说明) 和第 9 章。

### 4.5 翻译客户消息

如果客户发的是英文/中文你看不懂的消息，可以点消息下方的小字 **「翻译」**，系统会显示译文。

> 翻译用的是 **Google 翻译（免费）**，不消耗 AI 配额。

### 4.6 头像和状态显示说明

- **客户头像**：从 Instagram API 拉取，URL 有时效性，过期后图片加载失败会自动显示为**文字缩写**（比如 "MA"）
- **消息状态**：每条 AI 发的消息右下角会显示「已发送 AI」标签，区别于人工发送
- **未读小蓝点**：对话列表里有蓝点 = 客户最新消息你还没看

## 5. 评论收件箱

打开**评论收件箱**页面，能看到：

- 客户在你**任何帖子**下的评论
- 每条评论的处理状态：
  - **已自动回复**：触发了某条评论规则，系统已自动公开回复 + 发私信
  - **已跳过（功能关闭）**：你关闭了评论自动触发，所以系统只记录没回复
  - **未匹配**：评论里没有任何规则关键词
- 点「打开私信对话」会跳转到对应的私信会话（如果该客户已被私信触发）

> **想看客户评论在哪条帖子下？** 点评论行右侧的 🔗 链接，会跳转到 Instagram 原帖。

未读评论会在侧边栏显示红色数字徽标。

## 6. 评论触发规则

打开**评论规则**页面，可以配置「客户在帖子下评论某些关键词 → 自动公开回复 + 自动私信」。

**典型规则示例：**

```
规则名：询价触发
关键词：price, 多少钱, quote, 报价
匹配方式：包含
公开回复模板：Hi {{username}}! Thanks for your interest. I'll DM you the details now.
私信模板：Hi {{username}}, here's our pricing for delivery in Toronto: ...
```

**字段说明：**

| 字段 | 说明 |
|------|------|
| 名称 | 规则的备忘名（不会发给客户） |
| 关键词 | 多个关键词用英文逗号分隔，**任一**命中即触发 |
| 匹配方式 | `包含` / `精确匹配` / `正则表达式` |
| 公开回复模板 | 在评论区**公开**回复的内容；可用 `{{username}}` 占位符 |
| 私信模板 | 自动发送的**私信**内容；可用 `{{username}}` |
| 启用状态 | 关闭后此规则不再触发 |

**全局开关：** 仪表盘上有一个**评论触发功能**总开关，关掉之后所有规则都不生效，但评论仍然会被记录到收件箱。

## 7. 系统设置

打开**系统设置**页面。从上到下：

### 7.1 AI 模型选择

下拉菜单里分三组：

- **Anthropic**：Claude 系列（最专业、贵）
- **OpenAI**：GPT-5.4 / GPT-4.1 / o3 等（综合最强）
- **Google**：Gemini 系列（便宜、快）

**怎么选？**

| 场景 | 推荐 |
|------|------|
| 客户消息复杂、需要业务判断 | Claude Sonnet 或 GPT-5.4 |
| 业务简单、追求性价比 | gemini-2.5-flash 或 gpt-4.1-mini |
| 试水 / 调试 | gemini-2.5-flash-lite（最便宜） |

**自定义模型**：如果你想用 DeepSeek、xAI、Groq 等兼容 OpenAI 接口的服务：

1. 在「自定义模型」一行填入模型 ID（如 `deepseek-chat`）
2. 选择「OpenAI 兼容」
3. 点「使用」
4. 下方会出现 API Key 和 API 地址输入框，填入对应平台的信息

### 7.2 API Key 怎么填

**关键提醒：API Key 输入框现在是普通文本框（不是密码框），原因是浏览器密码管理器会**自动填充登录密码**到密码框，把正确的 API Key 覆盖掉。如果你的浏览器有密码管理器：

- ✅ 直接在框里粘贴正确的 key（`sk-...` 或 `sk-ant-...` 或 `AIza...`）
- ✅ 点击输入框外面（**失焦**）才会保存
- ❌ 不要让浏览器/密码管理器自动填充

**如果发现 AI 回复返回 401 或「Incorrect API key」错误**，多半是 key 被错误数据覆盖了，去这里重新填一次。

### 7.3 回复语言（翻译策略）

三个选项，控制**发出去的消息**要不要翻译：

| 选项 | 行为 |
|------|------|
| **自动检测**（推荐） | 客户用什么语言，回复用什么语言。检测到中英文不匹配时自动翻译 |
| **始终翻译为英文** | 不管 AI 生成什么语言，发出去都翻译成英文 |
| **不翻译** | 原样发送，AI 写啥就发啥 |

> 后台的**手动生成回复**永远是中文草稿（方便你审核），发送时按这个策略翻译。

### 7.4 回复延迟

数字（秒），AI 收到客户消息后等几秒再回复，**让对话看起来像真人在打字**而不是机器秒回。

- `0` = 立即回复（机器人感强）
- `3`（默认）= 3 秒后回复（自然）
- `10` = 10 秒后回复（更像在思考，但客户可能等不及）

### 7.5 欢迎语

- **新用户自动欢迎**开关：客户**首次**给你发消息时，先发一条欢迎语再让 AI 接管
- **欢迎语内容**：自定义文本，例如：

```
您好！感谢您联系 FleetNow Delivery，请问有什么可以帮到您？
```

### 7.6 新对话默认模式

新客户首次发消息时，对话默认进哪个模式：

- **AI 自动**：AI 接手所有新对话（推荐用于成熟业务）
- **我亲自接**：所有新对话都需要你手动处理（推荐用于初期或高客单价业务）

### 7.7 管理者偏好（重要功能）

这是 InstaBot 最特别的功能：**AI 会自动学习你的风格偏好**。

**怎么工作？** 你在生成 AI 回复时，每次写的「风格提示词」（比如「回复短一点」「不要用感叹号」「语气更亲切」），系统会用 LLM 分析这条提示词是不是**长期规则**：

- ✅ 长期规则：「不要用感叹号」「回复不超过3句话」「用专业的语气」→ 自动入库，以后所有回复都遵循
- ❌ 一次性指令：「这次写得详细一点」「针对这个具体问题再说说」→ 不入库

**怎么管理？**

- **查看**：在这个面板里能看到所有学到的偏好（如「使用简洁、专业的语气」）
- **停用 / 启用**：暂时关掉某条偏好（不删除）
- **删除**：彻底删掉
- **手动添加**：直接在输入框写一条规则，点「添加」

**实战建议：** 不需要一次写一长串规则——每次生成回复时顺手提一句你的要求，系统自己会归纳。用 1-2 周后偏好库会自然成型。

### 7.8 通知

- **新消息通知** 总开关
- **桌面通知**：浏览器弹窗（首次会要权限）
- **提示音**：声音提示
- **标签页标题闪烁**：你切换到别的标签页时，浏览器标题会闪 `(*)` 提醒你

## 9. 常见情况说明

### 9.1 消息发不出去（24 小时窗口）

报错：`This message is sent outside of allowed window`

**原因：** Instagram 平台规定，只有客户**在过去 24 小时内**给你发过消息，你才能用 API 回复。超过 24 小时未互动的客户，就算你在后台点发送也发不出去。

**解决：** 没办法，是 Meta 的硬性规则。需要等客户主动联系你。

### 9.2 AI 回复里不能带网址

报错：`Link can't be shared`

**原因：** Instagram 临时限制了你的账号发链接（防垃圾机制）。这个限制是 `is_transient: true`，**1-3 天会自动解除**。

**当前 AI 已设置不在回复中包含 URL**，但如果你手动发链接也会被拦。临时方案：让客户自己 Google 搜公司名。

### 9.3 头像加载失败显示文字

Instagram 头像 URL 有时效，过期后会自动显示为用户名缩写（如 "MA"）。这是正常现象，不影响发消息。

### 9.4 切到其他页面再回来加载慢

第一次回到对话页面时会比较慢，因为系统会顺便去 Instagram API 拉取没头像/没用户名的客户资料。每次最多拉 2 个，下次再拉剩下的，避免一次卡很久。多刷几次就齐了。

### 9.5 AI 突然不生成回复了

最常见原因（按概率排序）：

1. **API Key 错了**：浏览器自动填充把 key 覆盖成登录密码——去**系统设置**重新填
2. **API 配额用完**：去对应平台（OpenAI / Anthropic / Google）看账单
3. **后端服务挂了**：找运维 `systemctl status instabot`
4. **网络问题**：稍等几分钟重试

如果不确定原因，按 F12 打开浏览器控制台 → Network 标签，点一次「生成回复」，看红色失败的请求点开看 **Response**，里面会有具体报错信息。

### 9.6 想看 AI 用的是哪个模型

去**系统设置 → AI 模型**，下拉框里**已选中**的就是当前在用的。

### 9.7 想换更便宜的模型做翻译

目前**消息翻译**（点客户消息下方「翻译」）用的是免费 Google 翻译，不消耗 AI 配额。

**AI 翻译并优化**（人工面板里那个）用的是和生成回复同一个模型。如果想分开配置（生成用 GPT-5.4，翻译用 mini），找开发实现。

## 10. 工作流推荐

**新手日常：**

```
登录 → 私信对话页 → 看未读消息
   ├── 简单咨询：让 AI 自动回复（默认模式即可）
   ├── 复杂问题：切「我亲自接」→ 写中文草稿 → AI 翻译并优化 → 发送
   └── 重要客户：切「我亲自接」→ 全程手动跟进
```

**每周维护：**

- 看仪表盘的 AI 解决率，判断 AI 表现
- 去**评论收件箱**看有没有漏掉的评论
- 去**管理者偏好**看 AI 学到了哪些规则，删掉不准确的
- 如果客户问的问题 AI 频繁答不好，找开发更新知识库（`knowledge_base/*.md`）
